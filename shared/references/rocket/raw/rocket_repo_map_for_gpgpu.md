## 1. Repository Role

- `CONFIRMED`: Rocket Chip is a Scala/Chisel generator that emits RTL for a complete SoC, not just a CPU core. The README calls the repo a meta-repository, says hardware generation is a Scala program that emits RTL for a complete SoC, and lists packages spanning AMBA, devices, diplomacy, tilelink, system, unittest, and groundtest, not only `rocket` (`ref_submodule/rocket-chip/README.md:73-80`, `ref_submodule/rocket-chip/README.md:114-159`).
- `CONFIRMED`: elaboration is parameter-driven. `freechips.rocketchip.diplomacy.Main.elaborate` accepts `top` and `config`, instantiates a `RawModule` or `LazyModule`, emits FIRRTL plus annotations, and writes generated artifacts (`ref_submodule/rocket-chip/src/main/scala/diplomacy/Main.scala:13-18`, `ref_submodule/rocket-chip/src/main/scala/diplomacy/Main.scala:19-30`, `ref_submodule/rocket-chip/src/main/scala/diplomacy/Main.scala:41-52`).
- `CONFIRMED`: system variants are composed from `With...` config fragments. `BaseConfig`, `DefaultConfig`, `TinyConfig`, and others are simple `Config` compositions rather than separate handwritten SoC tops (`ref_submodule/rocket-chip/src/main/scala/system/Configs.scala:14-38`, `ref_submodule/rocket-chip/src/main/scala/system/Configs.scala:78-107`).
- `INFERRED`: for GPGPU skill extraction, the reusable value is generator structure, interface negotiation, system composition, MMIO/debug plumbing, and verification entry points; Rocket core execution internals are mostly context only for this shard (`ref/skillref/rocket.md:3-6`, `ref/skillref/rocket.md:23-29`, `ref/skillref/rocket.md:31-38`).

## 4. What Not To Transfer

- `CONFIRMED`: do not transfer Rocket’s five-stage scalar pipeline partition, decode/control logic, or frontend speculation structures. Those live under `ref_submodule/rocket-chip/src/main/scala/rocket/*` and are explicitly out of scope for this plan (`ref/skillref/rocket.md:31-38`; `ref_submodule/rocket-chip/src/main/scala/rocket/RocketCore.scala:158-220`; `ref_submodule/rocket-chip/src/main/scala/rocket/Frontend.scala:82-220`).
- `CONFIRMED`: do not copy CLINT/PLIC/debug register layouts verbatim. Transfer the attachment and declarative-MMIO pattern, not the CPU-specific ABI (`ref_submodule/rocket-chip/src/main/scala/devices/tilelink/CLINT.scala:94-101`; `ref_submodule/rocket-chip/src/main/scala/devices/tilelink/Plic.scala:195-220`; `ref_submodule/rocket-chip/src/main/scala/devices/debug/Debug.scala:254-260`).
- `CONFIRMED`: do not treat TileLink as mandatory for GPGPU. The transferable lesson is negotiated protocol capability plus adapters, evidenced by both AXI4<->TL directions (`ref_submodule/rocket-chip/src/main/scala/amba/axi4/ToTL.scala:64-73`; `ref_submodule/rocket-chip/src/main/scala/tilelink/ToAXI4.scala:31-35`; `ref_submodule/rocket-chip/src/main/scala/tilelink/ToAXI4.scala:96-129`).
- `CONFIRMED`: do not let local modules invent widths, IDs, cache policies, or address maps. Rocket centralizes them through config keys, topology objects, and negotiated edges (`ref_submodule/rocket-chip/src/main/scala/subsystem/Configs.scala:30-64`; `ref_submodule/rocket-chip/src/main/scala/tile/BaseTile.scala:49-64`; `ref_submodule/rocket-chip/src/main/scala/tilelink/Edges.scala:18-27`).
- `INFERRED`: do not transplant Rocket’s coherence assumptions blindly into a SIMT machine. Even Rocket warns some configs are only safe under restrictive conditions (`ref_submodule/rocket-chip/src/main/scala/subsystem/Configs.scala:183-205`).
